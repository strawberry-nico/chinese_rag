### 2. 问题分类与意图识别
**现状**：所有问题使用同一套处理流程
**需求场景**：
- 数值问题："营收是多少？"（需要精确数值）
- 布尔问题："是否盈利？"（需要Yes/No）
- 开放问题："发展如何？"（需要综合分析）
- 比较问题："A和B哪个更好？"（需要对比分析）

**实现建议**：
```python
class QuestionClassifier:
    def classify(self, query):
        """返回问题类型和处理策略"""
        categories = {
            'numeric': self._is_numeric_question(query),
            'boolean': self._is_boolean_question(query),
            'comparative': self._is_comparative_query(query),
            'open_ended': self._is_open_ended(query)
        }
        return max(categories, key=categories.get)
```

- "总结整个行业的发展趋势"

## ⚠️ 中优先级功能

### 4. 动态知识更新
**现状**：知识库静态，需要手动重新索引
**需求场景**：
- 定期更新的报告（季报、年报）
- 实时数据集成（股价、新闻）
- 增量索引而非全量重建

### 5. 个性化推荐
**现状**：所有用户得到相同结果
**需求场景**：
- 根据用户角色展示不同详细程度（高管vs分析师）
- 基于历史查询推荐相关内容
- 关注领域记忆（用户A关注财务，用户B关注技术）

### 6. 结果可信度评估
**现状**：没有答案可信度指标
**需求场景**：
- 标注答案置信度
- 标识信息来源的可靠性
- 提醒用户可能的遗漏或偏差

## 🔧 低优先级功能

## 🤔 思考与讨论

1. **对话管理复杂度**：是否需要维护完整的对话状态机？还是简单的上下文拼接？

2. **专业场景深度**：医疗/法律等专业场景需要多深的领域知识？是否超出RAG范围？

3. **实时性要求**：哪些功能需要实时响应？哪些可以异步处理？

4. **准确性vs灵活性**：专业场景下，高准确性要求是否会限制系统灵活性？

5. **用户群体定位**：面向专业用户还是普通用户？这会影响功能设计的复杂度。
